GOOGLE OAUTH - TROUBLESHOOTING

=== POSSÍVEIS PROBLEMAS E SOLUÇÕES ===

🔍 PROBLEMA 1: URLs não autorizadas
❌ Sintomas: Erro "redirect_uri_mismatch" ou "unauthorized_client"
✅ Solução: Verificar Google Cloud Console

No Google Cloud Console:
1. APIs & Services → Credentials → Seu OAuth Client
2. Authorized JavaScript origins:
   ✓ https://andrejulio072.github.io
   ✓ http://localhost:3000

3. Authorized redirect URIs:
   ⚠️ CRÍTICO: https://qejtjcaldnuokoofpqap.supabase.co/auth/v1/callback

🔍 PROBLEMA 2: Credenciais incorretas no Supabase
❌ Sintomas: Botão Google não responde ou erro genérico
✅ Solução: Verificar configuração no Supabase

No Supabase Dashboard:
1. Authentication → Providers → Google
2. Verificar se está "Enabled" ✓
3. Client ID: deve ser do Google Cloud Console
4. Client Secret: deve ser do Google Cloud Console

🔍 PROBLEMA 3: Google OAuth não habilitado
❌ Sintomas: Erro "access_denied" ou "OAuth not enabled"
✅ Solução: Habilitar Google+ API

No Google Cloud Console:
1. APIs & Services → Library
2. Buscar "Google+ API" ou "People API"
3. Clicar "Enable"

🔍 PROBLEMA 4: Domínio não verificado
❌ Sintomas: Tela de aviso "App não verificado"
✅ Solução: Adicionar usuários de teste

No Google Cloud Console:
1. OAuth Consent Screen
2. Test users → Add users
3. Adicionar seu email para testes
